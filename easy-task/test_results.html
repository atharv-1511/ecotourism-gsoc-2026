Question 1: How to match sightings with weather data?

The Challenge

Students need to understand how to join occurrence data with weather records. The key insight is understanding which rows are kept vs. discarded when using different join types.

Our Approach

  • Focus on Gouldian Finch sightings in the Northern Territory
  • Filter to September (end of dry season - peak birdwatching)
  • Use inner_join() to match on ws_id and date
  • Teach students that inner_join keeps only matching rows from both tables

Solution Implementation

We created Exercise 1 that asks students to join Gouldian Finch sightings with weather data and analyze temperature patterns.

# Exercise 1: Inner join sightings with weather gf_sept_weather <- gf_nt_sept |> inner_join( weather |> select(ws_id, date, temp, min, max, prcp, rh), by = c("ws_id", "date") ) # Key insight: some sightings may be lost if no weather record exists nrow(gf_nt_sept) # original sightings nrow(gf_sept_weather) # after join (may be fewer)

Question 2: How to add tourism region information?

The Challenge

The ecotourism package contains tourism_region data linking weather stations to tourism regions. How do we enrich our sightings data with region names while preserving all sightings?

Our Approach

  • Use left_join() to preserve all sightings
  • Join via ws_id to get region names
  • Demonstrate that left_join keeps all rows from the left table
  • Handle potential NA values for unmatched stations

Solution Implementation

Exercise 2 guides students through enriching the data with tourism regions and creating visualizations by region.

# Exercise 2: Left join with tourism regions gf_with_regions <- gf_sept_weather |> left_join( tourism_region |> select(ws_id, region_name, state), by = "ws_id" ) # All sightings preserved, region added where available nrow(gf_with_regions) == nrow(gf_sept_weather) # TRUE # Visualize by region gf_with_regions |> count(region_name) |> ggplot(aes(x = reorder(region_name, n), y = n)) + geom_col(fill = "#2d6a4f")

Question 3: How to find data quality issues?

The Challenge

Not all weather stations have complete records. How can we identify sightings that occurred at stations with no weather data? This is a real-world data quality check.

Our Approach

  • Use anti_join() to find non-matching rows
  • Identify sightings where weather data is unavailable
  • Teach students about data completeness checks
  • Show practical implications for analysis

Solution Implementation

Exercise 3 uses anti_join to perform data quality analysis, finding "orphan" sightings without weather records.

# Exercise 3: Anti join to find missing weather data missing_weather <- gf_nt_sept |> anti_join( weather, by = c("ws_id", "date") ) # These are sightings with no matching weather record cat("Sightings without weather:", nrow(missing_weather), "\n") # Which stations are affected? missing_weather |> count(ws_id, sort = TRUE) |> head(10)
